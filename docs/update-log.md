# 업데이트 로그

기온별플리 / By Degrees의 주요 변경사항을 날짜별로 정리합니다. 커밋 단위 전체 목록이 아니라 운영과 제품 관점에서 의미 있는 묶음 기준입니다.

## 2026-06-04

- 관리자 통계의 퍼널 기준을 명확히 분리했습니다.
  - 상단 KPI는 생성 완료 건수 기준으로 표시합니다.
  - 퍼널은 방문자 / 생성 완료 이용자 / 저장 시도 이용자 / 공유 이용자처럼 고유 세션 기준으로 표시합니다.
  - 방문 -> 생성 전환율은 생성 완료 건수가 아니라 생성 완료 고유 세션 기준으로 계산합니다.
- 인기 곡 TOP 10 집계를 곡 제목만이 아니라 provider track id 기준으로 분리했습니다.
  - 제목이 같아도 아티스트나 실제 트랙이 다르면 별도 곡으로 집계합니다.
  - 관리자 화면에서 인기 곡 제목 아래 아티스트명을 함께 표시합니다.
- `/preview`, `/en/preview`에도 방문 이벤트 트래킹을 추가했습니다.
  - 한국어 / 영어 미리보기 페이지 직접 방문도 방문자 지표에 포함됩니다.

## 2026-06-03

- 임시 음악 provider를 iTunes Search API 중심으로 전환했습니다.
  - iTunes 아티스트 / 곡 검색 provider를 추가했습니다.
  - 기존 Spotify route는 alias로 유지하고, 클라이언트는 provider-neutral `/api/music/*` route를 사용하도록 정리했습니다.
  - iTunes 검색 결과에 메모리 캐시와 Neon 캐시를 적용했습니다.
  - 검색 실패 시 14일 이내 stale cache fallback을 사용하도록 보강했습니다.
  - iTunes 검색 캐시 워밍 스크립트를 추가했습니다.
- 검색 오류 운영 대응을 개선했습니다.
  - 단일 검색 실패 한 번으로 바로 점검 공지를 켜지 않고, 같은 route/query/error 조합이 일정 횟수 이상 반복될 때만 랜딩 점검 공지를 켭니다.
  - 점검 공지가 켜질 때 Discord webhook으로 운영 알림을 보냅니다.
  - 관리자 화면에서 점검 공지를 수동으로 내릴 수 있게 했습니다.
- 점검 완료 알림 기능을 추가하고 운영 방식을 정리했습니다.
  - 랜딩 점검 공지 안에서 이메일로 점검 완료 알림 신청을 받을 수 있게 했습니다.
  - `maintenance_subscribers`에 이메일을 저장하고, 개인정보 안내 문구를 추가했습니다.
  - Resend 자동 발송 대신 관리자 화면에서 미발송 이메일 목록을 복사하고 Gmail 작성창을 BCC prefill로 열도록 변경했습니다.
  - 발송 완료 후 `notified_at`을 기록하는 수동 운영 흐름을 사용합니다.
  - 발송 완료 후 3일이 지난 알림 신청자는 Vercel Cron으로 삭제하도록 cleanup API를 추가했습니다.
- 관리자 대시보드를 개선했습니다.
  - 일일 / 주간 / 월간 탭을 누르면 각각 오늘 / 이번 주 / 이번 달로 자동 이동합니다.
  - 국가별 / 대륙별 방문자 수와 생성 완료 이용자 수를 donut chart로 표시합니다.
  - 완성 기준 인기 아티스트를 TOP 10으로 확장했습니다.
  - 검색 / 탐색 기준 통계를 제거하고 생성 완료 기준 지표에 집중했습니다.
  - 관리자 로고 클릭 시 `by-degrees.vercel.app`을 새 탭으로 열도록 했습니다.
- 한국어 / 영어 플로우를 추가했습니다.
  - `/en`, `/en/create`, `/en/preview` route를 추가했습니다.
  - 랜딩 좌상단에 `KOR | ENG` 토글을 추가했습니다.
  - 생성 플로우, 검색 모달, 미리보기, 공유 문구를 locale copy dictionary로 분리했습니다.
  - X 공유 해시태그는 한국어 / 영어 플로우 모두 동일한 정책을 사용합니다.
- 공유와 저장 UX를 다듬었습니다.
  - iTunes / Spotify 검색 결과 안내 문구를 필요에 따라 제거하거나 provider에 맞게 조정했습니다.
  - 랜딩 문의하기 mailto 버튼을 추가했습니다.
  - 랜딩 목업과 시작 버튼 간격을 조정했습니다.
- 운영 비용 대응을 진행했습니다.
  - 원격 앨범아트 이미지 최적화를 비활성화해 Vercel Image Optimization 사용량 초과를 줄였습니다.
  - favicon artwork를 교체했습니다.
- 문서를 현재 운영 상태에 맞게 갱신했습니다.
  - README, architecture, checklist, copy guide, MVP plan, design guide, env example을 iTunes 중심 운영 기준으로 업데이트했습니다.
